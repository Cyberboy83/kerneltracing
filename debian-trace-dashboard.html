eBPF / bpftrace

Linux perf_event subsystem profiler. CPU sampling, cache miss analysis, scheduler latency, lock contention, memory access patterns. Brendan Gregg's FlameGraph included.

Essential Commands
eBPF / bpftrace one-liners
# CPU stack trace profiling (99Hz) $ sudo bpftrace -e 'profile:hz:99 { @[kstack] = count(); }' # All syscalls by process $ bt-syscalls # Trace every file open $ bt-openat # Disk I/O latency histogram $ bt-biodist # OOM kill watcher $ bt-oomkill # TCP connection tracing $ bt-tcpconn # malloc size distribution $ bt-malloc
perf + FlameGraph
# Live CPU hotspot view $ perf-cpu # Full flamegraph (10s capture → SVG) $ perf-flame # L1/L2/LLC cache miss stats $ perf-cache # Scheduler wake-up latency $ perf-sched # Memory access patterns $ perf-mem # Custom flamegraph for PID $ flame 1234 /tmp/myapp.svg 30 # Lock contention analysis $ perf-lock
BCC Tools (60+ ready-to-use)
# Trace all new process executions $ bcc-execsnoop # File open tracing $ bcc-opensnoop # Block I/O latency histogram $ bcc-biolatency # Memory leak detection $ bcc-memleak # Off-CPU time profiling $ bcc-offcputime # TCP session lifespan $ bcc-tcplife # Linux capability check trace $ bcc-capable
ftrace / trace-cmd
# Start function tracer $ ftrace-start # Read trace output $ ftrace-read # List available events $ ftrace-events # Function graph recording $ trace-cmd record -p function_graph -g do_sys_open sleep 5 # View trace report $ trace-cmd report # GUI viewer $ kernelshark # Find function to probe $ kern-probe tcp_sendmsg
